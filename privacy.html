Rias Gremory Privacy Policy

Last updated: June 2026

This Privacy Policy explains what data the Rias Gremory Discord bot processes, why it is processed, how long it is kept, and how users can request deletion.

1. Overview

Rias Gremory is a multi-purpose Discord bot currently developed and operated for a specific Discord community server. The bot provides server management, AI chat, AI image/GIF understanding, administrator-only voice AI, moderation tools, support tickets, leveling, level roles, welcome/goodbye messages, rank cards, and leaderboards.

The bot is configured by authorized server administrators through an administrator dashboard. Regular users cannot configure privileged bot features themselves.

This bot is not designed for advertising, user profiling, selling data, or unrelated analytics. Data is processed only to provide the bot’s intended Discord server features.

2. Data the Bot May Process

Depending on which features are enabled by server administrators, the bot may process the following types of Discord data:

  • Discord user IDs
  • Discord server IDs
  • Discord channel IDs
  • Discord message IDs
  • Message content in enabled AI channels
  • Message content inside ticket channels when transcript logging is enabled
  • Message attachments, such as images or GIFs, when used with AI image/GIF understanding
  • XP and level data
  • Voice XP data
  • Moderation records, such as warnings, mutes, bans, reasons, timestamps, target user IDs, and moderator IDs
  • Ticket metadata, such as ticket owner ID, ticket channel ID, open time, close time, and transcript file information
  • Member join and leave events for welcome/goodbye features

The bot does not collect real names, passwords, payment information, private Discord account credentials, or any data unrelated to the bot’s server features.

3. AI Chat

AI chat is only active in channels that are explicitly enabled by server administrators. If AI chat is not enabled in a channel, the bot does not use that channel’s messages for AI chat responses.

When AI chat is enabled, the bot reads user message content in that enabled channel to generate conversational AI responses. This is necessary for the bot to understand what the user asked and respond appropriately.

AI chat uses temporary, user-specific conversation memory. This memory is limited to recent conversation context, approximately the last 6–8 messages, and is automatically cleared after about 30 minutes of inactivity from that user.

AI chat memory is not used for advertising, user profiling, or training models by the bot owner.

4. AI Image and GIF Understanding

The bot may process images or GIFs sent by users when they ask the AI to understand or respond to visual content. This may include message attachments sent in AI-enabled channels.

Image and GIF data is processed only to answer the user’s request. The bot does not provide public AI image generation.

5. Voice AI

Voice AI is restricted to users with Administrator permission. Regular users cannot start the voice AI feature.

Voice AI only runs when an authorized administrator explicitly starts it. When active, the bot joins a voice channel and processes voice input transiently to generate an AI audio response.

The bot does not store audio files. The bot does not keep persistent voice transcripts. Speech is processed only temporarily for the purpose of generating a response.

The bot may display operational logs such as connection status, usage estimates, or error messages, but it is not intended to log the actual spoken words or AI response text persistently.

6. Ticket System and Transcripts

The ticket system allows users to open private support channels through a button-based ticket panel. Ticket channels are visible only to the ticket creator, the bot, and staff members with the required moderation or management permissions.

If ticket transcript logging is enabled by server administrators, the bot may save a text transcript of the ticket channel when the ticket is closed. Ticket transcripts are used for support review, moderation safety, abuse prevention, and staff accountability.

Users are informed inside the ticket channel that the ticket may be recorded for support and moderation safety.

Ticket transcript files are stored locally by the bot owner. Transcript file names include the ticket owner’s Discord user ID and the ticket channel ID so that deletion requests can be handled correctly.

Ticket transcripts are automatically deleted after 30 days. Users may also request earlier deletion through the bot’s official support server.

Ticket transcripts are not used for advertising, user profiling, or model training by the bot owner.

7. Leveling and XP Data

The bot includes text XP and voice XP systems. These systems may store user IDs and XP/level values so users can view their rank cards, leaderboards, and progression.

Server administrators may configure level roles. When a user reaches a configured level, the bot may use member information to assign the appropriate role.

Leveling data is used only for community progression features and is not used for advertising or profiling.

8. Moderation Data

The bot includes moderation tools such as warnings, mutes, bans, and moderation case logging.

Moderation records may include:

  • The target user’s Discord user ID
  • The moderator’s Discord user ID
  • The moderation action type
  • The reason provided by staff
  • The timestamp of the action

Moderation data is stored so staff can review previous moderation history, enforce server rules consistently, and prevent abuse.

9. Welcome and Goodbye Features

The bot may process member join and leave events to send configured welcome or goodbye messages. These messages may include basic server and member information, such as the server name, member count, username, display name, or user mention.

This data is used only to provide welcome/goodbye functionality.

10. Administrator Dashboard

The bot’s settings are managed through an administrator dashboard. The dashboard is intended for authorized administrators only.

Administrators may use the dashboard to configure AI channels, ticket settings, transcript settings, welcome/goodbye settings, leveling settings, and other server features.

Regular users cannot use the dashboard to enable privileged features or change bot configuration.

11. Use of Third-Party AI Services

Some AI features may require sending relevant input, such as message content, image attachments, or temporary voice-derived text/audio data, to third-party AI API providers in order to generate responses.

The bot owner does not use user message content, ticket transcripts, images, or voice input to train machine learning or AI models.

AI data is transmitted only as needed to provide the requested AI functionality.

12. What the Bot Does Not Do

Rias Gremory does not:

  • Sell user data
  • Use data for advertising
  • Use data for unrelated analytics
  • Use message content to profile users
  • Store Discord account passwords or login credentials
  • Store audio files from voice AI
  • Keep persistent voice transcripts
  • Provide public AI image generation
  • Request or use Presence Intent to track online status or activities

13. Data Retention

Different types of data are kept for different lengths of time depending on the feature:

  • AI chat memory is temporary and clears after about 30 minutes of user inactivity.
  • Ticket transcripts are automatically deleted after 30 days.
  • XP and level data may be kept while the leveling system is active.
  • Moderation records may be kept as long as necessary for server safety and staff review.
  • Voice AI audio files and persistent voice transcripts are not stored.

14. Data Deletion Requests

Users may request deletion of stored data through the bot’s official support server.

For ticket transcript deletion requests, the ticket owner’s Discord user ID may be used to locate transcript files. Transcript files are automatically deleted after 30 days even if no manual deletion request is made.

Some moderation records may need to be retained for server safety, abuse prevention, or rule enforcement. Deletion requests will be reviewed based on the type of data and the reason it is stored.

15. Data Security

The bot owner takes reasonable steps to protect stored bot data. Access to bot files, dashboard configuration, and stored transcript files is limited to authorized operators.

Users should not share sensitive personal information, passwords, payment information, private credentials, or other highly sensitive data with the bot or inside ticket channels.

16. Children and Sensitive Information

Users should not intentionally provide sensitive personal information to the bot. The bot is intended to operate within Discord servers according to Discord’s own rules and age requirements.

If sensitive information is accidentally shared in a ticket or message, users may request deletion through the official support server.

17. Changes to This Policy

This Privacy Policy may be updated when the bot’s features, data handling, or retention practices change. The “Last updated” date at the top of this page will be changed when the policy is updated.
